Brians Games Internal Guidlines

Quality Assurance

7/24/19 Every game should aim for the following features to uphold high Quality

  • is good
  • gamepad support if applicable
  • touchscreen and mobile support if applicable
  • consistency across multiple devices and browsers
  • Options to customize sound, music, and controls for accesibility
    • Need a library to handle this to add to all games
    • Controls customization is the only new thing here
  • Use pre-processed sounds. runtime modification of sound effects should not be allowed because it is inconsistent across development and publication
  • runs on multiple browsers
  • runs on multiple devices
  • customizable control schemes
  • links to the site [More Games]
  • Has Brians Games Branding

Dev Standards

  • Separate Development and Production builds
  • Don't use runtime sound effect modifiers
  • Add google analytics
  • Add Meta tags for SEO
  • Use basic DRM warning

Shared Engine Template

A basic engine can contain the following features that would shared between all game projects

  • Input Handling
    • Gamepad
    • Touch
    • maybe also keyboard and mouse
